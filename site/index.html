How It Works

Three steps to structured AI

Copy the stack, fill in your config, and point Claude Code at it. That's the whole setup.

01

Copy the stack

Clone ApexStack into your project as a directory. It's just markdown files.

cp -r apexstack/ .apexstack/
02

Configure

Edit onboarding.yaml with your company name, team members, tech stack, and workflow preferences.

$EDITOR .apexstack/onboarding.yaml
03

Use it

Reference the stack from your CLAUDE.md. Claude Code now knows your team structure and processes.

@.apexstack/CLAUDE.md
onboarding.yaml
company: name: "Acme Corp" mission: "Making widgets simple" team: - name: "Alice" role: "tech-lead" # maps to roles/engineering/tech-lead.md - name: "Bob" role: "backend-engineer" - name: "Charlie" role: "product-manager" tech_stack: language: "TypeScript" framework: "Next.js" database: "PostgreSQL" quality: test_coverage_target: 80 required_reviews: 1

Quality Gates

Every phase transition has a gate. PRD approved before design. Tests passing before review. Code review before merge. QA before done. No exceptions.

Code Review

Structured process with architecture, security, testing, and performance checklists. Severity levels for feedback. Response time targets.

Deployment

CI/CD pipeline stages, environment promotion, pre-deploy checklists, rollback procedures, and post-deploy monitoring guidance.

Decision Records

Agent Decision Records (AgDR) capture technical choices made by AI agents -- what was considered, what was chosen, and why.
